##周五佳接口文档

###获取最佳创意列表展示的时候应该
1. 有创意用户（或许有头像）
2. 有创意发布时间
3. 有创意的title
4. 有创意的内容（包括文字和上传图片）
5. 有点赞数，和点踩数。

###请求类型
GET
###返回样例
 { 
  略
 }

###返回字段说明

| 字段 | 数据类型 | 说明 | 是否可空（默认为False）|
| ---- | :----: | :---------------: | ----------------: | 
| content        | varchar    	    | 创意内容 | 
| id	  	| int 	           | 创意编号 |
| author      	  | varchar	        | 用户（单点登录，数据库保存用户数据么？不知道）|
| support 	   | int	            | 点赞数 |
| nonsupport	 | int	            | 点踩数 |
| publish_time   | unix时间戳 	| 发布日期 |
| images | array| 创意上传的图片        | true |
| externalurl	| url | （没看懂pdf里面上传创意时候外部链接是干吗的）|  ture  |


###备注(如果需要)：
/?week=X中的X为请求的周数，比如第二周为/?week=2
